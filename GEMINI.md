## Git 변경사항 분석 및 커밋 가이드

다음 작업을 순차적으로 수행해주세요:

1. **변경사항 확인**
   - `git status` 결과를 확인하고, 각 파일의 수정/추가된 부분을 분석해주세요
   - 가능하다면 `git diff` 내용도 함께 검토해주세요

2. **파일 그룹화 및 커밋 메시지 제안**
   - 변경된 파일들을 기능/목적별로 논리적으로 묶어서 그룹화
   - 각 그룹별로 한국어 커밋 메시지 작성 (Conventional Commits 형식)
   - 출력 형식:

```
     그룹 1:
     파일: 파일1, 파일2, 파일3
     커밋 메시지: feat: ㅇㅇㅇ 기능 추가

     그룹 2:
     파일: 파일4, 파일5
     커밋 메시지: fix: ㅇㅇㅇ 버그 수정
```

3. **브랜치명 추천**
   - 변경 내역의 주요 목적을 기반으로 적절한 브랜치명 제안
   - 형식: `feat/기능명`, `fix/버그명`, `refactor/리팩토링-내용` 등
   - 예: `feat/next-auth-integration`

4. **PR 문서 작성**
   - **현재 브랜치 → develop 머지용 PR**: 제목 및 상세 설명
   - **develop → main 머지용 PR**: 제목 및 설명
   - 아래 PR 템플릿 형식을 따라 작성

5. **리팩토링 문서 작성 (리팩토링 작업인 경우)**
   - 변경사항이 리팩토링 성격이면, 아래 리팩토링 템플릿에 맞춰 문서 작성
   - 사용자가 명시적으로 요청한 경우에도 작성

모든 내용은 한국어로 작성해주세요.

## PR 템플릿

### 제목

[분류] 간결한 변경 내용 요약

예: `[FE] Next-Auth 인증 도입 및 서비스 레이어 리팩토링`

### 본문

#### 🎯 주요 변경 사항

- 변경 사항 1
- 변경 사항 2
- 변경 사항 3

#### 📝 상세 설명

이번 PR에서 수행한 작업에 대한 상세 설명

#### 🔍 변경 이유

왜 이 작업이 필요했는지 배경 설명

#### ✅ 테스트 방법

1. 테스트 단계 1
2. 테스트 단계 2

#### 📌 참고 사항

- 주의해야 할 점
- 관련 이슈/PR 링크
- 추가 작업 필요 사항

#### 📸 스크린샷 (선택)

UI 변경이 있는 경우 스크린샷 첨부
